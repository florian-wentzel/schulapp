# Informationen zu app_info.json
"latest_version" = neuste Version 

"warnings" ist wie folgt aufgebaut:

\"\< [x.x.x]" Alle Versionen vor x (inklusive) 

\"\> [x.x.x]" Alle Versionen nach x (inklusive)

\"\<\> [x.x.x] [y.y.y]" Alle Versionen von x bis y (inklusive)

Leerzeichen können nach dem kleiner/größer Zeichen weggelassen werden, ist aber schöner so..
